# Essential to know
- JWT is self-contained (stateless)
- Three parts: header.payload.signature
- Signed, not encrypted (don't store secrets)
- Good for APIs and microservices

# JWT (JSON Web Tokens)

## What is JWT?

JWT (pronounced "jot") is an open standard for securely transmitting information between parties as a JSON object. It's commonly used for authentication in modern web applications.

### Why JWT?

**Stateless:** The server doesn't need to store session data. All information is in the token itself.

**Scalable:** Perfect for microservices and distributed systems.

**Cross-domain:** Works across different domains and services.

**Mobile-friendly:** Easy to use with mobile apps.

## JWT Structure

A JWT consists of three parts separated by dots:

```
xxxxx.yyyyy.zzzzz
↑       ↑      ↑
header  payload signature
```

### 1. Header

Contains metadata about the token:

```json
{
  "alg": "HS256",  // Algorithm used
  "typ": "JWT"     // Token type
}
```

Base64Url encoded:
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
```

### 2. Payload (Claims)

Contains the actual data:

```json
{
  "sub": "1234567890",    // Subject (user ID)
  "name": "John Doe",     // User name
  "email": "john@example.com",
  "role": "user",         // User role
  "iat": 1516239022,        // Issued at (timestamp)
  "exp": 1516242622         // Expiration (timestamp)
}
```

**Standard Claims:**
- `sub` (subject) - User identifier
- `iss` (issuer) - Who issued the token
- `aud` (audience) - Who the token is for
- `exp` (expiration) - When token expires
- `iat` (issued at) - When token was created
- `nbf` (not before) - Token not valid before this time

### 3. Signature

Ensures the token hasn't been tampered with:

```
HMACSHA256(
  base64UrlEncode(header) + "." +
  base64UrlEncode(payload),
  secret
)
```

## Complete JWT Example

```
// Header
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
.
// Payload
eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ
.
// Signature
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

Decoded:
```javascript
// Header
{"alg":"HS256","typ":"JWT"}

// Payload
{
  "sub": "1234567890",
  "name": "John Doe",
  "iat": 1516239022
}
```

## Implementing JWT in Node.js

### Installation

```bash
npm install jsonwebtoken
```

### Generating Tokens

```javascript
import jwt from 'jsonwebtoken';

const JWT_SECRET = process.env.JWT_SECRET;  // Keep this secret!
const JWT_EXPIRES_IN = '15m';  // Short-lived

// Generate access token
function generateAccessToken(user) {
  return jwt.sign(
    {
      sub: user.id,
      email: user.email,
      role: user.role
    },
    JWT_SECRET,
    {
      expiresIn: JWT_EXPIRES_IN,
      issuer: 'myapp.com',
      audience: 'myapp.com'
    }
  );
}

// Usage
const token = generateAccessToken(user);
// eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

### Verifying Tokens

```javascript
function verifyToken(token) {
  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    return decoded;
  } catch (error) {
    if (error.name === 'TokenExpiredError') {
      throw new Error('Token expired');
    }
    if (error.name === 'JsonWebTokenError') {
      throw new Error('Invalid token');
    }
    throw error;
  }
}

// Middleware example
function authenticateToken(req, res, next) {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];  // Bearer TOKEN
  
  if (!token) {
    return res.status(401).json({ error: 'Access token required' });
  }
  
  try {
    const user = verifyToken(token);
    req.user = user;
    next();
  } catch (error) {
    return res.status(403).json({ error: error.message });
  }
}
```

### Login Route with JWT

```javascript
app.post('/auth/login', async (req, res) => {
  const { email, password } = req.body;
  
  // Validate credentials
  const user = await db.findUserByEmail(email);
  if (!user || !await bcrypt.compare(password, user.passwordHash)) {
    return res.status(401).json({ error: 'Invalid credentials' });
  }
  
  // Generate tokens
  const accessToken = generateAccessToken(user);
  const refreshToken = generateRefreshToken(user);
  
  // Store refresh token (database or Redis)
  await db.saveRefreshToken(user.id, refreshToken);
  
  res.json({
    accessToken,
    refreshToken,
    tokenType: 'Bearer',
    expiresIn: 900  // 15 minutes in seconds
  });
});
```

## Access Tokens vs Refresh Tokens

### The Problem

If you use one long-lived token:
- Security risk if stolen
- Can't revoke without changing secret
- No way to force logout

### The Solution: Token Pair

```
┌─────────────────┐     Short-lived (15 min)     ┌──────────────┐
│  Access Token   │ ───────────────────────────→ │  API Calls   │
│  (In memory)    │                              │  Protected   │
└─────────────────┘                              └──────────────┘
         │                                               │
         │ Expired                                       │ 401
         │                                               │
         ▼                                               ▼
┌─────────────────┐     Long-lived (7 days)        ┌──────────────┐
│  Refresh Token  │ ───────────────────────────→   │  Auth Server │
│  (HttpOnly      │                              │  New Access  │
│   Cookie)       │ ←───────────────────────────   │  Token       │
└─────────────────┘                              └──────────────┘
```

### Implementation

```javascript
// Generate refresh token (longer-lived)
function generateRefreshToken(user) {
  return jwt.sign(
    { sub: user.id, type: 'refresh' },
    JWT_REFRESH_SECRET,  // Different secret!
    { expiresIn: '7d' }
  );
}

// Refresh endpoint
app.post('/auth/refresh', async (req, res) => {
  const { refreshToken } = req.body;
  
  if (!refreshToken) {
    return res.status(401).json({ error: 'Refresh token required' });
  }
  
  try {
    const decoded = jwt.verify(refreshToken, JWT_REFRESH_SECRET);
    
    // Check if token is in database (allows revocation)
    const storedToken = await db.getRefreshToken(decoded.sub);
    if (storedToken !== refreshToken) {
      return res.status(403).json({ error: 'Invalid refresh token' });
    }
    
    // Get user and generate new tokens
    const user = await db.findUserById(decoded.sub);
    const newAccessToken = generateAccessToken(user);
    const newRefreshToken = generateRefreshToken(user);
    
    // Rotate refresh token (security best practice)
    await db.saveRefreshToken(user.id, newRefreshToken);
    
    res.json({
      accessToken: newAccessToken,
      refreshToken: newRefreshToken
    });
  } catch (error) {
    return res.status(403).json({ error: 'Invalid refresh token' });
  }
});
```

## Secure Token Storage

### In Web Applications

**Option 1: HttpOnly Cookies (Recommended)**

```javascript
// Server: Send tokens as cookies
res.cookie('accessToken', accessToken, {
  httpOnly: true,
  secure: true,
  sameSite: 'strict',
  maxAge: 15 * 60 * 1000  // 15 minutes
});

res.cookie('refreshToken', refreshToken, {
  httpOnly: true,
  secure: true,
  sameSite: 'strict',
  maxAge: 7 * 24 * 60 * 60 * 1000  // 7 days
});
```

**Option 2: Memory Only (SPA with API)**

```javascript
// Store in JavaScript memory only
let accessToken = null;

function setToken(token) {
  accessToken = token;
}

function getToken() {
  return accessToken;
}

// On page reload, use refresh token to get new access token
// (refresh token in HttpOnly cookie)
```

**❌ Never store in:**
- LocalStorage (XSS vulnerable)
- SessionStorage (XSS vulnerable)
- Plain cookies without httpOnly

### In Mobile Apps

```javascript
// React Native example
import * as SecureStore from 'expo-secure-store';

// Store securely
await SecureStore.setItemAsync('accessToken', token);

// Retrieve
const token = await SecureStore.getItemAsync('accessToken');

// Delete on logout
await SecureStore.deleteItemAsync('accessToken');
```

## Token Expiration & Rotation

### Short-Lived Tokens

```javascript
// Access tokens: 15 minutes
// Refresh tokens: 7 days

const TOKEN_CONFIG = {
  access: { expiresIn: '15m' },
  refresh: { expiresIn: '7d' }
};
```

### Refresh Token Rotation

```javascript
// When refreshing, invalidate old refresh token
app.post('/auth/refresh', async (req, res) => {
  const { refreshToken } = req.body;
  
  // Verify old token
  const decoded = jwt.verify(refreshToken, JWT_REFRESH_SECRET);
  
  // Check if token exists in whitelist
  const tokenData = await db.getRefreshTokenData(decoded.sub);
  
  if (tokenData.token !== refreshToken) {
    // Possible token reuse attack!
    await db.revokeAllUserTokens(decoded.sub);
    return res.status(403).json({ error: 'Security violation detected' });
  }
  
  // Generate new pair
  const user = await db.findUserById(decoded.sub);
  const newAccessToken = generateAccessToken(user);
  const newRefreshToken = generateRefreshToken(user);
  
  // Replace old token with new one
  await db.replaceRefreshToken(decoded.sub, refreshToken, newRefreshToken);
  
  res.json({ accessToken: newAccessToken, refreshToken: newRefreshToken });
});
```

## JWT Security Best Practices

### 1. Use Strong Secrets
```bash
# Generate secure secret
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```

### 2. Keep Tokens Short-Lived
```javascript
// Access tokens: 5-15 minutes
// Refresh tokens: 1-7 days
```

### 3. Don't Store Sensitive Data
```javascript
// ❌ BAD
const token = jwt.sign(
  { password: user.password, ssn: user.ssn },
  secret
);

// ✅ GOOD
const token = jwt.sign(
  { userId: user.id, role: user.role },
  secret
);
```

### 4. Validate All Claims
```javascript
jwt.verify(token, secret, {
  issuer: 'myapp.com',
  audience: 'myapp.com',
  algorithms: ['HS256']  // Only allow specific algorithms
});
```

### 5. Handle Token Errors Properly
```javascript
try {
  const decoded = jwt.verify(token, secret);
} catch (error) {
  if (error.name === 'TokenExpiredError') {
    // Prompt for refresh
  } else if (error.name === 'JsonWebTokenError') {
    // Malformed token - possible attack
    logSecurityEvent('invalid_token', token);
  }
}
```

## Common JWT Mistakes

### 1. Using JWT for Sessions
JWTs are not sessions! Don't store them in a database and look them up - that defeats the purpose.

### 2. Storing Too Much Data
JWTs are sent with every request. Keep them small (< 8KB).

### 3. Ignoring Algorithm Confusion
```javascript
// ❌ VULNERABLE - accepts any algorithm
jwt.verify(token, secret);

// ✅ SECURE - specify allowed algorithms
jwt.verify(token, secret, { algorithms: ['HS256'] });
```

### 4. Storing Secrets in Code
```javascript
// ❌ BAD
const secret = 'my-secret-key';

// ✅ GOOD
const secret = process.env.JWT_SECRET;
```

### 5. Not Validating Issuer/Audience
Always validate tokens are meant for your application.
